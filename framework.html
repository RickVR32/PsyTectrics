01 — Structure

Two layers.
One system.

Every person operates across two simultaneous awareness layers — conscious and subconscious. The framework reads both, measures the gap between them, and produces a unified profile from the interaction.

Layer 01 — Conscious

The presented
self.

The external, observable layer. What a person shows — their behaviour, decisions, communication, and patterns under normal conditions. The conscious layer is what people believe about themselves.

  • 16 Source Code questions
  • 8 App integration scores
  • 4 SC Blueprint™ direct questions
  • Observable behaviour and output
  • Self-reported awareness state
Layer 02 — Subconscious

The operating
layer.

The internal, hidden layer running simultaneously. What drives the conscious layer without the person's awareness — fear, memory, desire, instinct, attachment. Scored through reversed polarity questions.

  • 16 reversed Source Code questions
  • 8 subconscious App scores
  • 4 subconscious SC Blueprint™ questions
  • Emotional patterns and loops
  • The inverse awareness state
The gap — Drift

The distance
between layers.

Drift measures the divergence between what a person consciously presents and what is actually running underneath. A person can appear perfectly regulated while running a destabilising subconscious programme. Drift catches it.

Low Drift (below 1.0) means the layers are largely aligned — what you show is close to what runs. High Drift (above 2.5) means there is significant divergence — a gap between the presented self and the operating self that is likely producing cost, friction, or loops the person doesn't fully see.

Drift = |Conscious average − Subconscious average|

Always present.

03 — Scoring Engine

Hierarchical.
Not flat.

The Awareness Architecture uses a hierarchical scoring engine — not a flat average. Source Codes feed into Apps. Apps feed into SC Blueprints™. Each layer inherits and refines the layer below it.

Layer 1 Source Codes 28 con · 28 sub · scored 1–5
↓ average of 2 Source Codes
Layer 2 App Scores 8 conscious · 8 subconscious
↓ 70% apps · 30% direct OC question
Layer 3 Observer Code Final integrated score
↓ proportional share of total
Output OC Ratio 25/25/25/25 = Zero-Point

Why hierarchical?

A flat scoring system treats every question with equal weight. The Awareness Architecture treats questions as building blocks — two Source Codes produce an App score, two Apps produce an SC Blueprint™. Each level of aggregation dampens outliers and produces a more structurally honest result.

The 70/30 split between hierarchical App scores and direct SC Blueprint™ questions means the score is both bottom-up (built from granular Source Code data) and top-down (validated by direct gut-check responses). Where these diverge significantly, the Drift reading increases.

The integration model

The OC ratio uses an integration model — not a dominance model. A higher score means more of that code is expressed and available, not more activated or problematic. 25/25/25/25 at full integration is the peak — all four codes equally expressed and accessible. This means higher scores are always positive, and the goal is equal distribution, not suppression of any code.

Score band reference
1–2 — Activated · high load · cost or blind spot
3   — Balanced · moderate · functional
4–5 — Regulated · resource · integrated
